# セイリ部 画像制作エンジン

完成済みの記事Markdownを入力にして、AI画像生成プロンプト、図解レイアウト仕様、ローカル合成指示、画像制作プランを生成するワークフローです。

標準の完成フローはCanvaではなく、`seiribu-editorial/scripts/compose_eyecatch.py` によるローカル合成です。生成素材にタイトル、サブタイトル、セイリ部ロゴを正確に合成して完成画像を作ります。

Canvaは任意の手動微調整ツールとして扱います。チームCanvaとCodex接続Canvaが違う場合があるため、Canva API連携を本番完了条件にしてはいけません。

記事内にアイキャッチ指定がない場合でも、汎用アイキャッチを勝手に補完してはいけません。足りない指定として品質チェックに出し、記事固有の検索意図に合わせたブリーフを追加してから制作します。

## 起動方法

```bash
python3 seiribu-editorial/scripts/generate_image_plan.py \
  seiribu-editorial/drafts/pillars/sell-or-dispose-clean.md
```

## 【アイキャッチと本文図解の作り分け】

**1. アイキャッチ素材（ローカル合成前提）**
- アイキャッチ用画像は完成品として出力せず、**「背景なしのイラスト素材（または切り抜きやすいシンプルな背景）」**として生成します。
- タイトル文字、サブタイトル、ロゴ、背景の合成は、後工程でローカル合成を使って行います。AI側でアイキャッチ用のタイトル文字や余白（文字パネルなど）を生成する必要はありません。

**2. 本文図解・情景イラスト**
- 情景イラストや写真風素材は、文字なし・ロゴなしのビジュアル素材として画像生成AIで作ります。
- 日本語ラベル、見出し、比較表、フロー、矢印を含む本文図解は、画像生成AIに丸投げしません。素材だけを生成し、Pillow・SVG等で正確にレイアウト生成します。

## 【基本方針とトーン】
- **画風（アートスタイル）**: 温かみのあるフラットイラスト（主線なし）、または日本の現代的な日常を表現した高品質なベクターアート。写真風にする場合は、生々しすぎない明るいトーン。
- **トーン**: やさしい、清潔、少し生活感がある。不安を煽らない。50代以上にも親しみやすい。
- **テーマ**: おしゃれな収納写真より、実家の荷物、段ボール、古い物、確認や仕分けの状況が伝わる画像。読者が「これは自分の状況に近い」と感じることを優先する。
- **図解と情景のバランス**: B2B向けのコンサル資料ではないため、図解（比較表やフロー）だけでなく、雰囲気を伝える情景イラストもバランスよく配置する。ただし親子・家族は記事の検索意図に必要な場合だけ使う。

## 【避けたい画像（NG表現）】
- 真っ白で生活感がないミニマリスト部屋
- 高級感だけが強い買取業者風の画像
- 汚部屋、ゴミ屋敷感が強すぎる画像
- 暗い遺品整理、孤独、恐怖を煽る画像
- 現金や高額査定を強調する画像

## 【画像サイズ（アスペクト比）と制作物の状態管理】

AI画像生成ツール（`generate_image`等）はシステムの仕様上、**正方形（1:1）しか生成できない**前提でワークフローを構築すること。
- **アイキャッチ用素材**: `1:1`（正方形）で「文字なし・背景なし」の純粋なイラスト素材として生成し、後工程（ローカル合成やCanva等）で1.91:1や16:9の横長キャンバスに配置・合成する。
- **記事内図解・情景イラスト**: `1:1`（正方形）で素材を生成し、後工程でレイアウトする。
- **プロンプトの注意点**: ツール側でアスペクト比指定（--ar 16:9 等）を受け付けないため、プロンプトにはアスペクト比の指示を含めないこと。

生成した画像は、以下のステータスで厳格に管理・保存する。
- `source/`: 生成直後や背景透過前の元素材
- `draft/`: 採用前の候補
- `final/`: QAを通過し、記事やWordPressで使う確定画像
- `rejected/`: 不採用画像（公開用の `assets/images/` 等には絶対に入れない）

## 【アイキャッチのブランド型とロゴ配置（後工程用）】
※AIは画像生成時にこれらを描画しない。最終組み立て時の仕様。
- **ローカル合成テンプレ**: `seiribu-editorial/scripts/compose_eyecatch.py` を使用。
- **構成**: ロゴは左上、見出しは上部中央に大きく、サブタイトルは見出し直下。ビジュアル素材は下部から中央へ配置。
- **フォント**: 見出しはUDモトヤアポロ太字、サブタイトルはNoto Sans JP Regular。
- **ロゴ元画像**: `assets/images/brand/seiribu-logo.png`
- **ロゴ必須**: アイキャッチ完成版と本文図解完成版には、セイリ部ロゴを必ず小さく配置する。
- **注意点**: 見出し、図解ラベル、人物の顔、重要な品物の上にロゴを重ねない。画像全体の主役は記事内容にし、ロゴは小さなブランド署名として扱う。

## 【展開・派生の考え方】
- 同じ人物配置、同じ箱分け、同じ背景を別記事へ貼り替えるだけの流用は禁止。
- 同じブランドトーンを保ちながら、記事固有の主役（本、カメラ、着物など）に合わせて描き分けること。

## 【完了条件】
- **本エンジンの基本動作**: AIツール（`generate_image`等）を用いてAI自身が自律的に画像を生成してはいけません。Markdown/JSONの制作プラン（外部生成AI用プロンプト、図解レイアウト仕様、ローカル合成指示）を出力した時点で「完全停止（完了）」してください。
- 画像生成までの依頼: ※ユーザーから「画像を実際に生成して」と明示的に指示された場合のみ実行。生成候補を `draft/` に保存し、文字混入・ロゴ混入・構図流用を確認したうえで、採用品をローカル合成し、完成画像だけを `final/` に昇格して完了。
- Canva仕上げまでの依頼: ユーザーが明示した場合のみ、Canvaプラグインまたは人間のCanva作業で微調整する。Canvaを使えない場合でも、ローカル合成の完成画像があれば標準完了。
