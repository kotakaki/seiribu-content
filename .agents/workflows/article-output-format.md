# 記事出力フォーマットルール

記事を `.md` ファイルとして出力する際、以下のフォーマットに従うこと。
目的は「WordPressへのコピペ入稿時に、不要な情報を消す手間をゼロにする」こと。

## ⛔ 最重要ルール

1. **1ファイル＝1記事**。設計用ファイルと入稿用ファイルを分けない。
2. **本文中に「PHASE」「H2-1」等の制作用ラベルを絶対に書かない。**
3. ファイル冒頭のHTMLコメント（凍結済み設計図）は、WordPress上では非表示になるため、消す必要がない。
4. 本文はHTMLコメントの直後から始め、**そのままコピペで入稿できる状態**にすること。

## ファイル構成

```
<!-- === 凍結済み設計図（Invariants） === ... === END === -->

# タイトル（H1）

導入文

## H2見出し（番号なし）

本文...

## よくある質問

Q. 質問文
A. 回答文

## まとめ

本文...
```

## 凍結済み設計図のフォーマット

HTMLコメント `<!-- === 凍結済み設計図（Invariants） === ... === END === -->` の中に、以下の【】セクションを含めること。

### 必須セクション

```
【記事基本情報】
- メインKW: 
- スラッグ: 
- Meta Description: 
- 記事タイプ: 

【採用タイトル】
（確定したタイトル1本）

【ターゲット読者】
（誰の・どんな悩みを解決するか）

【検索意図】
（読者が何を知りたくて検索しているか）

【Out of Scope（扱わないこと）】
- （この記事では扱わないトピック）

【見出し構成案とインサイト】
H2: 見出しテキスト
  H3: 見出しテキスト
  H3: 見出しテキスト
H2: 見出しテキスト
...

【まとめ・CTA設計案】
まとめ：（要点の方向性）
CTA：（誘導先がある場合）
```

### 任意セクション（必要に応じて追加）
- 【内部リンク設計】
- 【競合との差別化ポイント】
- 【画像設計】

## 本文のルール

### 導入文（対象読者の吹き出し＋この記事でわかること）
- 導入文の最後には必ず、セイリ部部長のKackyによる対象読者への呼びかけ（吹き出し）と、「この記事でわかること」の箇条書きをセットで配置する。
- 以下のフォーマットに厳密に従うこと。

```markdown
🗨️吹き出し（Kacky）：
（対象読者に向けた共感や、どんな人に役立つかの一言。例：「実家じまいで悩んでいる方は、この記事が参考になります！」）

**この記事でわかること**
- 箇条書き1
- 箇条書き2
- 箇条書き3
- 箇条書き4
```

### 見出し
- `##` はH2、`###` はH3で出力。番号（①②、H2-1等）は付けない。
- 読者が「何が書いてあるか」をひと目で判断できる見出しにする。

### 内部リンク
- `> **あわせて読みたい：**` の引用ブロック形式で、説明1文＋リンクを配置。

### 画像作成指示
- 本文中の該当箇所に引用ブロック形式で配置：
```
> [!CMS担当者へ：ここに図解を挿入]
> 種類：（横型フロー図 / 比較表 / 概念図 など）
> ALT：（代替テキスト）
> 指示：（具体的な配置・要素・表現の指示）
```

### FAQ
- `Q. **質問文**` と `A. 回答文` のペアで書く。
- 質問文は太字にする。
- 構造化データ（schema.org FAQPage）に変換可能な形式を維持。

### 禁止事項
- 本文中に `---`（水平線）を使わない。
- 本文末尾にメタ情報を別途書き出さない（凍結済み設計図に集約する）。
- frontmatter（YAMLの`---`ブロック）は入稿用ファイルには使わない。
