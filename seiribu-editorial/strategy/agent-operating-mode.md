# セイリ部 Codex運用方針

## 現在の運用モード

このリポジトリのCodexは、当面「記事執筆エージェント」としては使いません。

記事本文の生成は精度が安定せず、チャットも重くなりやすいため、ここでは以下の2つを主な役割にします。

1. サイト運営の進捗管理
2. 完成した記事に使う画像制作・画像指示の作成

## 担当すること

### 進捗管理

- タスクの整理
- 優先順位の整理
- Notionタスク化の補助
- 記事ラインナップ、カテゴリ、グロナビ、内部リンク方針の管理
- Search ConsoleやAnalyticsの結果を見た改善タスクの整理
- 公開前チェックリストの作成
- 作業ログ、方針変更、判断メモの保存

### 画像制作

- 完成記事の内容に合うアイキャッチ画像案
- 記事内画像案
- 画像生成プロンプト
- Canva用タイトル、サブコピー
- WordPress画像タイトル
- ALTテキスト
- キャプション候補
- ファイル名
- 画像設置位置の提案

## 原則として担当しないこと

- 記事本文の新規執筆
- H2ごとの本文生成
- 導入文の生成
- FAQ本文の生成
- 既存記事の全文リライト
- 記事の量産

記事本文が必要な場合は、人間が書いた完成稿、または別工程で作成した原稿を入力として扱います。

Codexはその原稿に対して、画像化しやすい場面の抽出、画像タイトル・ALT作成、進捗管理、チェックリスト化を担当します。

## 例外

ユーザーが明示的に依頼した場合のみ、短い文言案や見出し案、メタディスクリプション案などの補助テキストを作ってよいです。

ただし、主力記事やピラー記事の本文をこのチャット内で一括生成しません。

## 作業時の出力先

- 運営方針: `strategy/`
- 進捗管理、工数管理、レポート: `reports/`
- 画像方針、画像プロンプト: `strategy/image-agent-workflow.md` または `assets/images/`
- 完成記事から作った画像指示: `assets/images/<article-slug>/README.md`

## 確認フロー

画像制作では、以下の順番で進めます。

1. 完成記事または記事URLを確認する
2. 画像の役割を決める
3. 画像案、タイトル、ALT、Canva文言を作る
4. 必要なら画像生成またはCanva編集を行う
5. 人間が最終確認する

公開操作は人間が行います。
