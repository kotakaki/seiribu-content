# セイリ部 画像生成・仕上げワークフロー

## 結論

画像生成用のエージェントは作った方がよいです。

現在のセイリ部Codexは、記事本文を書くエージェントではなく、完成した記事に画像を付けるためのエージェントとして運用します。

画像は記事本文とは別工程にして、記事ごとに以下を作ります。

- アイキャッチ画像の目的
- 読者に伝えたい状況
- 避けたい表現
- 画像概要
- 画像生成AI用プロンプト、または図解レイアウト仕様
- ファイル名
- WordPress画像タイトル
- ALTテキスト
- キャプション候補
- 設置位置
- 後工程で配置するタイトル、サブタイトル、ロゴ
- Canvaを使う場合の任意微調整指示

記事にアイキャッチ指定がない場合でも、画像制作エンジンが汎用アイキャッチを自動生成してはいけません。足りない指定として品質チェックに出し、記事ごとの検索意図に合わせたブリーフを追加してから制作します。

## 現在の既定運用

`scripts/generate_image_plan.py` の既定モードは `standard` です。

standardモードでは、アンチグラビティの記事内CMSブリーフをもとに、次の基本セットを出力します。

- アイキャッチ用画像素材: 1件
- 記事内画像・図解: 合計4件
- Canva仕上げ指示
- 画像生成AI用プロンプト
- 本文画像・図解のロゴ配置指示

記事内のCMSブリーフは、画像の挿入位置、ALT、キャプション候補をWordPress投稿時に使うため残します。ただし、英語の完成プロンプトやDALL-E指定は記事本文には残さず、画像制作プラン側へ分離します。

軽量確認だけをしたい場合は、以下のように `light` モードを指定します。

```bash
python scripts/generate_image_plan.py --mode light drafts/example.md
```

## 役割分担

| 役割 | 担当すること | 担当しないこと |
| --- | --- | --- |
| 記事エージェント | 記事内に画像概要、設置位置、ALT候補、避けたい表現を書く | 画像制作手法の最終判断、完成プロンプトの固定 |
| セイリ部Codex / 画像制作エンジン | 進捗管理、画像タイプ判定、画像生成AI用プロンプト化、図解レイアウト仕様化、構図、トーン、NG表現、画像タイトル、ALT | 記事本文の新規執筆、全文リライト |
| Codex画像仕上げ | 本文画像・図解の生成、ロゴ配置、完成画像出力 | 記事本文の意味づけ |
| Canva仕上げ | アイキャッチのタイトル、サブタイトル、ロゴ配置、Canvaフォントでの最終調整 | 本文画像・図解の標準仕上げ |
| 人間 | 最終確認、違和感チェック、公開判断 | なし |

## 基本方針：記事内の指定は「制作ブリーフ」として扱う

記事内に書く画像指定は、完成プロンプトではなく制作ブリーフです。

記事エージェントは、本文理解に必要な画像の目的と概要までを書きます。画像制作エンジンは、その概要をもとに、アイキャッチ、アイキャッチ合成用素材、情景イラスト、写真風素材、記事内図解、図解用小物素材のどれで作るべきかを判断し、必要に応じてプロンプトを書き換えます。

既存記事の画像や前回生成した候補は、参考にしてもそのまま使いません。構図、人物、品物、背景のうち何を記事固有に変えたかを確認してから制作します。

日本語ラベル入りの図解は、ChatGPTの画像生成に近い自然な文字描画を使うため、短いラベルや見出しまで `imagegen` で生成します。後工程では原則として図解文字を重ねず、セイリ部ロゴだけを小さく正確に合成します。

## Canva連携の現在地

標準のアイキャッチ完成フローは、Canva手動仕上げです。Canvaでしか使えないフォントを使うため、Codexは文字・ロゴなしのアイキャッチ素材まで自動生成し、タイトル、サブタイトル、セイリ部ロゴの最終合成は人間がCanvaで行います。

Canvaは任意の手動微調整ツールとして扱います。チームCanvaとCodexコネクタのCanvaアカウントが違う場合、API連携は本番完了条件にしません。

Codex環境にはCanvaプラグインがあり、Canvaのテンプレート検索、デザイン作成、既存デザイン編集、画像差し替えなどは別工程として実行できます。ただし、`scripts/generate_image_plan.py` はCanvaを直接操作しません。このスクリプトが作るのは、画像生成プロンプト、イラスト図解ベース指示、ロゴ合成指示、任意のCanva仕上げ指示までです。

主に以下の用途に向いています。

- ローカルで作った完成PNGを人間がCanvaへアップロードして微調整する
- 既存のCanvaテンプレを人間が複製する
- ページやデザイン内容を確認する
- 画像サイズを変える
- 複数記事のアイキャッチを同じデザインルールで揃える

## おすすめ運用

### 1. Canvaテンプレと本文画像仕上げルールで共通化する

アイキャッチはCanvaテンプレを標準にし、本文画像・図解はCodex側の自動仕上げルールを使います。

必要な要素:

- セイリ部ロゴ
- 記事タイトル枠
- 短い補足文枠
- 画像エリア
- 色とフォントの固定

### 2. 記事ごとに画像指示を作る

完成記事を確認したあと、本文とは別に画像指示を作ります。

```text
記事タイトル:

画像の目的:

読者に伝えたい状況:

入れたい要素:

避けたい要素:

画像概要（記事エージェントが書く制作ブリーフ）:

画像タイプ候補:

画像生成AI用プロンプト / 図解レイアウト仕様:

ファイル名:

WordPress画像タイトル:

ALTテキスト:

キャプション候補:

設置位置:

アイキャッチ用ロゴ配置位置:

仕上げタイトル:

仕上げサブ:
```

### 3. 画像素材を生成する

標準の画像生成ツールはCodexの `imagegen` スキルです。アイキャッチ素材と本文イメージは文字・ロゴなしで作ります。本文図解は、短い日本語ラベルや見出しまで `imagegen` で自然に描画し、後工程ではブランド帰属ロゴだけを小さく重ねます。

Antigravityから `image-briefs/<slug>-image-plan.json` をCodexへ渡された場合、Codexは `seiribu-image-finisher` スキルを使います。この場合は画像プランの確認だけで止めず、`imagegen` による画像生成、本文画像・図解のロゴ合成、`assets/images/<slug>/` への保存、README更新までを一続きの完了条件にします。

生成候補は、最初から `assets/images/...` に置きません。記事スラッグごとに `/private/tmp/seiribu-image-work/<article-slug>/` を作業置き場として使い、QAを通った採用画像だけを `assets/images/<article-slug>/` に移します。

生成候補が複数できた場合、公開用フォルダへ残すのは採用品だけです。比較用の旧版、背景抜き前の素材、クロマキー素材、不採用品は `/private/tmp/seiribu-image-archive/` へ退避します。退避前には必ずドライランで確認します。

```bash
python scripts/clean_image_assets.py --article <article-slug> --dry-run
```

実際に退避するときだけ、以下を実行します。

```bash
python scripts/clean_image_assets.py --article <article-slug> --apply
```

画像生成AIは、アイキャッチ素材と本文イメージでは背景、人物、物、小物アイコンなどのビジュアル素材だけを作ります。本文図解では、短い見出し・ラベル・矢印まで含めて生成してよいです。ロゴだけは生成AIに描かせず、後工程で合成します。

ユーザーが「Canvaで整える」「背景なし」「画像素材だけ」と指定した場合も、生成AIにはタイトル、ロゴ、不要な文字を入れない素材だけを作らせます。アイキャッチ完成版のタイトルとロゴはCanvaで人間が入れます。本文画像・本文図解は、ブランド帰属表示としてロゴを必ず後工程で小さく配置します。

セイリ部では以下のトーンを守ります。

- 日本の実家らしさ
- 記事で必要な場合だけ親子・家族の確認作業
- 古い物、趣味品、コレクション
- 明るく清潔
- 不安を煽らない
- 買取業者っぽくしすぎない
- 既存記事と同じ構図や素材を使い回さない

標準画風は、温かみのあるフラット編集イラスト、主線なしまたは控えめな主線、ベクター風、淡い色、清潔だが少し生活感のある日本の住まいです。写真風素材は、品物の実物感が必要な場合だけ使います。

画像生成AIへ渡すプロンプトでは、ピクセルサイズではなく `16:9`、`1:1` などのアスペクト比を指定します。記事に出す画像とアイキャッチ素材は原則 `16:9`、図解に添える単体小物素材だけ `1:1` を使います。`1200 x 675` や `1024 x 1024` は生成後の書き出し・リサイズ基準です。

### 3-1. 画像制作エンジンの判断フロー

記事内の画像概要を受け取ったら、以下の順番で判断します。

1. 画像の役割を判定する
   - アイキャッチ
   - アイキャッチ合成用素材
   - 記事内イメージ
   - 記事内図解
   - 図解用小物素材
   - 写真風素材
2. 画像制作方法を選ぶ
   - 情景イラストや写真風素材は画像生成エンジンで作る
   - 日本語ラベルや比較表を含む図解は、短い文字まで画像生成AIで自然に描かせ、後工程ではロゴだけを小さく重ねる
   - アイキャッチは、文字・ロゴなしの画像生成素材を作り、Canvaテンプレで仕上げる
   - アイキャッチ合成用素材は、タイトル、ロゴ、背景なしの切り抜き素材として作る
   - 図解用小物素材は、完成図解に添える単体アイコンや物だけを画像生成で作る
3. 記事内の概要を画像生成AI用プロンプトまたは図解レイアウト仕様に変換する
   - 読者層、場所、人物、品物、表情、光、避けたい表現を補う
   - アイキャッチ素材と本文イメージでは画像内文字、ロゴ、看板、透かしを禁止する。図解では短い日本語ラベルだけ許可する
   - アイキャッチではCanvaでタイトルやロゴを置くための余白を確保する
4. 完成後に確認する
   - 文字崩れがない
   - ロゴを入れる場合は、見出し、人物の顔、重要な品物を邪魔していない
   - 暗い遺品整理、ゴミ屋敷、高額買取広告のように見えない
   - 前の記事や別記事で使った構図の焼き直しになっていない
   - ファイル名、ALT、キャプション、設置位置をREADMEに残す

### アイキャッチの判断基準

アイキャッチは、検索ユーザーが「これは自分の状況に近い」と感じることを優先します。

基本は、実家の室内で古い物、段ボール、写真、本、カメラ、家具などを落ち着いて確認している編集イラスト調にします。親子や家族を入れるのは、記事の検索意図や本文で家族関係が主題になる場合に限ります。

写真風にするのは、実物感を出す必要が強い場合だけです。写真風は暗さ、汚さ、業者広告感が出やすいため、清潔で明るい生活感に寄せます。

アイキャッチのブランド型は、Canva側で左上にセイリ部ロゴ、上部中央に大きな見出し、その直下にサブタイトル、下部に人物や状況イラストを置く構成を基本にします。画像生成AIには、文字やロゴを描かせず、Canvaで置ける余白のある素材だけを作らせます。

フォントはCanvaの最終仕上げで指定します。見出しはUDモトヤアポロ太字、サブタイトルはNoto Sans JP Regularを使います。制作環境にUDモトヤアポロがない場合は、Noto Sans JP Boldなどで暫定確認し、最終Canvaで差し替えます。

### 記事内図解の判断基準

次の条件に当てはまる場合は、画像生成AIに完成図解を丸投げせず、イラスト図解ベースと後工程のラベル配置に分けます。

- 日本語の見出し、表、ラベルを正確に入れる必要がある
- 手順や比較を読ませることが目的である
- 3分類、4ステップ、費用比較など、情報構造が主役である
- 文字崩れがSEO記事の信頼性を下げる可能性がある

抽象的な線、丸、四角、矢印だけの図解は不可です。図解でも、人物、品物、実家の部屋、段ボール、着物、本、カメラなど、記事文脈に合う具体物を入れ、サンプルのようにイラストとして成立させます。

### 記事エージェントが本文内に書く指定の最小形

```text
> [!CMS担当者へ：ここに画像を挿入]
> 種類：アイキャッチ素材（Canva仕上げ用） / 記事内イメージ / 写真風素材 / 記事内図解 / 図解用小物素材
> 目的：この画像で読者に伝えたいこと
> ALT：画像を見られない人にも伝わる説明
> 指示：入れたい要素、避けたい要素、必要なラベル
```

この指定だけで足りない場合は、画像制作エンジン側で補います。ただし、アイキャッチの有無は勝手に補完しません。記事エージェントは、画像生成AI用プロンプトや図解レイアウト仕様を過度に固定しません。

### 4. 後工程で整える

生成画像をそのまま使うより、後工程で必要な最小限の統一感を出します。本文図解では原則として文字を後乗せせず、ロゴ合成だけに留めます。

- 画像の上にタイトルを載せる
- アイキャッチ完成版にはCanvaでセイリ部ロゴを入れる。本文画像・本文図解にも、ブランド帰属表示としてセイリ部ロゴを必ず小さく入れる
- 図解文字は画像生成AIに任せ、文字修正が必要な場合だけ例外的に手動調整する
- 余白を揃える
- 記事カテゴリごとに色を少し変える

## 画像生成エージェントのルール

### 必ず守ること

- 実家整理の読者に近い生活感を出す
- 40代から60代の子供世代を想定する
- 親の物を尊重している雰囲気にする
- 「捨てる」より「確認する」「分ける」を見せる
- アイキャッチ素材と本文イメージでは、画像生成AIに文字、日本語ラベル、英字、数字、ロゴ、透かし、看板、吹き出しを描かせない。本文図解では短い日本語ラベルと必要最小限の説明文だけ許可する
- アイキャッチのロゴとタイトルはCanvaで人間が最終合成する
- 本文画像・本文図解はブランド帰属表示としてロゴ必須。背景が薄い場合は透過ロゴ、濃い・複雑な場合は薄い白背景付きロゴを使う
- 既存記事の画像、前回生成した画像、別記事の構図を使い回さない
- 不採用画像は公開用の `assets/images/...` に残さない

### 避けること

- ゴミ屋敷のように汚く見せる
- 高額査定や札束を強調する
- 怖い遺品整理の雰囲気にする
- ミニマリスト風の真っ白な部屋にする
- 若者向けのポップすぎるデザインにする
- ロゴを大きく置きすぎる、見出し・人物の顔・重要な品物に重ねる
- 同じ箱分け、同じ親子配置、同じ部屋背景を記事タイトルだけ変えて再利用する

## 制作物の状態管理

| 状態 | 置き場所の考え方 | READMEや制作プランへの記載 |
| --- | --- | --- |
| source | 生成直後、背景透過前、編集前の素材 | 原則記載しない |
| draft | 採用前の候補。作業用フォルダか一時領域に置く | 原則記載しない |
| final | 記事やWordPressで使う確定画像。`assets/images/...` に置く | 記載する |
| rejected | 不採用画像。公開用フォルダから外し、必要なら `/private/tmp` へ退避 | 記載しない |

画像生成AIの出力は、最初から `final` に置きません。まず `source` または `draft` として扱い、文字混入、ロゴ混入、背景混入、既存画像との構図流用を確認したうえで、合格したものだけ `final` に昇格します。最終的に公開フォルダに残すのは、記事で使う画像だけです。候補画像や不採用画像を残すと、次回のエージェントがそれを採用品と誤認しやすくなります。

公開用フォルダに残す画像は、各 `assets/images/<article-slug>/README.md` の「推奨画像」または「完成画像」に載せる画像を基準にします。`scripts/clean_image_assets.py` は、この採用品セクションに載っていない画像を退避候補として検出します。README内の素材画像も一時的に残したい場合は、`--keep-all-readme-references` を付けて慎重に確認します。

## 生成前QA

画像を作る前に、最低限以下を確認します。

- 今回の記事固有の検索意図が画像に出ているか
- 既存記事と同じ構図、人物、品物、背景になっていないか
- アイキャッチ合成用素材の場合、ロゴ、タイトル、背景、床、壁、影を入れない指示になっているか
- 画像生成AIに渡す指示がピクセルサイズではなくアスペクト比になっているか
- 記事内図解の場合、日本語ラベルが短く、画像生成AIで自然に読める形になっているか
- 記事内図解が抽象的な線や図形だけでなく、サンプルのようなイラスト図解として成立しているか
- 画像制作プラン、README、実ファイル名が食い違っていないか

## 生成後QA

画像を作った後に、最低限以下を確認します。

- 既存記事の画像と並べて、同じ構図の流用に見えないか
- アイキャッチ合成用素材の場合、背景が透明または確実に除去しやすい単色になっているか
- アイキャッチ合成用素材の場合、タイトル、ロゴ、背景小物が画像に混ざっていないか
- 不採用候補が `assets/images/...` に残っていないか
- READMEには採用画像だけが載っているか

## 完了条件

画像制作プラン作成だけを依頼された場合は、Markdown/JSONの制作プラン、画像生成AI用プロンプト、図解レイアウト仕様、Canva仕上げ指示、本文画像・図解のロゴ配置指示が揃っていれば完了です。

lightモードで作った画像制作プランは、採用画像と保留画像が分かれていれば完了です。保留画像まで作る必要がある場合は、`--mode standard` で再生成してから進めます。

画像生成まで依頼された場合は、生成候補を `draft` として保存し、生成後QAを通したうえで、本文画像・図解はブランド帰属ロゴを合成し、完成画像を `final` に移します。アイキャッチはCanva仕上げ用素材までをCodex側の完了範囲にします。

Canva仕上げまで依頼された場合のみ、Canvaプラグインまたは人間のCanva作業でテンプレ複製、画像差し替え、タイトル配置、セイリ部ロゴ配置、保存まで完了したときだけCanva作業完了です。Canvaを実際に操作していない場合は、完了報告で「アイキャッチ素材まで」または「Canva任意微調整指示まで」と明記します。

## 画像タイトル・ALT生成ルール

画像を作るときは、必ず以下も同時に作ります。

| 項目 | ルール |
| --- | --- |
| ファイル名 | 英数字、ハイフン区切り。記事スラッグと用途を入れる |
| WordPress画像タイトル | 画像の内容がわかる日本語。記事タイトルの丸写しにしない |
| ALTテキスト | 画像を見られない人にも内容が伝わる説明。80字前後まで |
| キャプション | 必要な場合だけ。本文の理解を助ける短い説明 |
| 設置位置 | アイキャッチ、見出し直下、表の前後などを明記 |
| ロゴ配置位置 | アイキャッチ完成版はCanva側で左上を基本にする。本文画像・図解は右下・左上など、本文や図解ラベルを邪魔しない余白を明記 |

### ALTで避けること

- キーワードを不自然に詰め込む
- 「画像」「イラスト」だけで終わる
- 記事タイトルをそのまま繰り返す
- 画像にない情報を書く
- 「高く売れる」など根拠のない表現を入れる

### ALTの良い例

実家のリビングで親子が段ボールの荷物を確認し、古い本や写真を整理している様子

### ALTの悪い例

実家の整理 売れるもの 買取 処分 おすすめ

## 仕上げテンプレの推奨サイズ

- アイキャッチ: 1200 x 675
- 正方形SNS用: 1200 x 1200
- 記事内画像・記事内図解: 1200 x 675

2. Canvaアイキャッチテンプレ
3. 使用フォントと色
4. 画像サイズ
5. 本文画像・図解のロゴ配置ルール
6. カテゴリごとの色ルール

## 最初のおすすめ

まずは、仮ピラー4本だけCanvaテンプレと本文画像仕上げルールで揃えます。

その後、主力記事リライト時にアイキャッチを更新します。

最初から全記事の画像を作り直すと工数が大きくなるため、以下の順番がよいです。

1. 仮ピラー4本
2. 実家の片付けで売れるもの一覧
3. 買取基準に満たないとは？
4. 信頼できる買取業者の見分け方
5. 悪質な訪問買取業者の特徴
