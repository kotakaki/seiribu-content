# セイリ部 記事制作フロー

## 現在の扱い

この文書は、記事制作が必要になった場合の保管用フローです。

現在のCodex運用では、このチャットを記事本文の執筆エージェントとして使いません。主な役割は、サイト運営の進捗管理と、完成した記事に使う画像制作・画像指示の作成です。

本文、導入文、H2本文、FAQ本文の新規生成や全文リライトは、ユーザーが明示的に依頼した場合のみ行います。

## 目的

記事を一気に量産せず、検索意図、構成、本文、最終調整を段階的に確認しながら作ります。

特にセイリ部では、心理・家族の悩み、実家整理、買取判断が混ざりやすいため、構成承認前に本文を書き始めると記事が薄くなります。

このフローは、記事制作をステートマシンとして扱います。各フェーズの冒頭で現在フェーズを宣言し、フェーズ外の内容は出しません。

精度を保つため、本文作成前に検索意図とトピッククラスタ境界を必ず確認します。軽量フローでも、この最低ゲートは省略しません。

Codex実行時の詳細手順は、`.codex/skills/seiribu-seo-editor/` の `seiribu-seo-editor` スキルを参照します。この文書は人間が確認する制作フローの要約です。

記事制作の成果物は、原則として `briefs/` または `drafts/` のファイルに保存します。チャットには、保存先、要点、確認事項だけを返します。ユーザーが明示しない限り、本文・構成・FAQなどの長文をチャットに丸ごと貼りません。

## 制作モード

### 標準フロー

対象:

- ピラー記事
- Search Consoleで流入がある既存記事
- 心理・家族の悩み系の主力記事
- 買取判断、業者選び、収益導線に近い記事

原則:

- 構成承認までは本文を書かない
- 導入文はH2本文とは別フェーズで作る
- 本文はH2ごとに作る
- 各フェーズで停止し、人間の確認を待つ

### 軽量フロー

対象:

- 短い補助記事
- FAQ記事
- 用語整理記事
- 既存記事への内部リンク用の小記事

原則:

- 記事企画と構成をまとめて出してよい
- ただし本文作成前に構成確認を入れる
- 公開前チェックは省略しない

## PHASE 0: 検索意図とトピック境界

目的は、本文や見出しを作る前に、記事の守備範囲を決めることです。

出すもの:

```md
【現在実行中のフェーズ: PHASE 0】
KW:
検索意図:
主トピック:
同一クラスタ:
隣接クラスタ:
別トピッククラスタ:
扱わないこと:
本文前に注意すること:
```

ピラー記事、既存流入記事、収益に近い記事では、読者が最初に知りたいこと、既存記事の課題、参照した内部資料も確認します。

ここでは本文も見出し構成も書きません。

停止文:

> この前提で記事企画へ進めてよいですか？

## PHASE 1: 記事企画

目的は、記事の役割と検索意図を決めることです。

出すもの:

- 対象KW
- 想定読者
- 検索意図
- 読者の不安
- 記事の役割
- 扱わない範囲
- トピッククラスタ境界表
- 内部リンク候補
- 制作モード

トピッククラスタ境界表では、必ず以下を分けます。

| 区分 | 意味 |
| --- | --- |
| 主トピック | この記事で厚く書く中心 |
| 同一クラスタ | このピラー配下、または強く内部リンクする記事 |
| 隣接クラスタ | 軽く触れて内部リンクする程度 |
| 別トピッククラスタ | 原則本文で厚く扱わない |
| 書かないこと | 検索意図を濁すため除外する話 |

ここでは本文を書きません。

停止文:

> この企画で見出し構成へ進めてよいですか？

## PHASE 2: 見出し構成

目的は、記事の設計図を固めることです。

出すもの:

- H1案
- タイトル案
- メタディスクリプション案
- H2、H3構成
- 各H2で答えること
- 各H2の役割ラベル
- 判断表、チェックリスト、FAQ案
- 内部リンクとCTA
- 画像候補、ALT方針

各H2には、必ず以下のどれかの役割ラベルを付けます。

| 役割 | 書き方 |
| --- | --- |
| 主役 | 検索意図の中心。厚く書いてよい |
| 補助 | 主役を判断しやすくする説明。中程度に書く |
| 導線 | 別記事へ送る入口。短く書く |
| 注意書き | 誤解やトラブル防止。1段落または表の1項目に留める |
| 削除候補 | 検索意図から外れるため構成に入れない |

ここでも本文は書きません。

停止文:

> この構成で導入文制作へ進めてよいですか？

承認後のファイル操作:

- 構成案を下書きファイルに保存する
- ファイル冒頭にHTMLコメントで「凍結済み設計図」を残す
- 凍結済み設計図には、検索意図、主トピック、扱わないこと、H2役割ラベルを含める
- 以後の本文は、この凍結済み設計図から外れないようにする

## PHASE 3: 導入文制作

目的は、読者を記事本文へ自然に入れることです。

導入文はH2-1とは別に作ります。

導入文に入れるもの:

- 読者の状況
- 読者の不安
- 記事の結論
- この記事でわかること
- どの順番で整理すればよいか

導入文で避けること:

- 煽り
- 高額買取の断定
- いきなり業者へ誘導すること
- H2-1と同じ内容の繰り返し

停止文:

> この導入文でH2ごとの本文作成へ進めてよいですか？

## PHASE 4: H2ごとの本文作成

目的は、見出しごとに本文の精度を上げることです。

原則として、1回に書くのは1つのH2だけです。

各H2に入れるもの:

- セクション冒頭の結論
- 読者の不安への回答
- 実務上の注意点
- 必要に応じた表、箇条書き、チェックリスト
- 次に読む記事への自然な内部リンク候補

H2ごとの制限:

- 主役H2は厚く書いてよい
- 補助H2は必要な説明に絞る
- 導線H2は短くし、内部リンク先へ送る
- 注意書きはH2化しないか、1段落に留める
- 別トピッククラスタをH2本文として展開しない

停止文:

> このH2の内容で次のH2へ進めてよいですか？

## PHASE 5: 統合・最終調整

目的は、記事全体を公開前の下書きに整えることです。

確認するもの:

- タイトルと本文の一致
- 冒頭で検索意図に答えているか
- H2同士の重複
- 内部リンクの自然さ
- アフィリエイト誘導の強さ
- 画像タイトル、ALT、キャプション
- メタディスクリプション
- FAQ
- 公開前チェック

停止文:

> この内容をWordPress下書き化してよいですか？

## 重要ルール

- 主力記事は一括生成しない
- 構成承認前に本文を書かない
- 導入文はH2本文とは別に作る
- 心理記事は共感だけで終わらせず、実家整理、価値確認、買取・処分判断へ接続する
- 商標記事、口コミ記事、レビュー記事は実体験や独自情報がない限り量産しない
- 公開操作は人間が行う
