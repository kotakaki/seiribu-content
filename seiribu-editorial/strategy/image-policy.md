# セイリ部 画像方針

## 基本方針

画像は、最初から作り込みすぎなくてよいです。

仮ピラーでは、読者が「これは自分の状況に近い」と感じることを優先します。おしゃれな収納写真より、実家の荷物、段ボール、古い物、確認や仕分けの状況が伝わる画像が向いています。

画像制作エンジンの標準モードでは、アイキャッチ用素材1枚と記事内画像・図解4枚を基本セットとして扱います。軽量確認だけをしたい場合は `light` モードで絞れますが、通常の記事制作では `standard` を使います。

記事ごとの検索意図、読者の困りごと、本文の役割に合わせて画像テーマを決めます。既存記事の画像、構図、人物配置、品物配置をそのまま流用することは禁止です。似たテーマを扱う場合も、何を変えたかを制作メモに残します。

## 画像で伝えること

- 実家の荷物を整理している
- 捨てる前に価値を確認している
- 必要な記事では親子・家族で話し合っている
- 売る・残す・処分するに分けている
- 不安を煽らず、落ち着いて進められる

## 避けたい画像

- 真っ白で生活感がないミニマリスト部屋
- 高級感だけが強い買取業者風の画像
- 汚部屋、ゴミ屋敷感が強すぎる画像
- 暗い遺品整理、孤独、恐怖を煽る画像
- 現金や高額査定を強調する画像

## 仮ピラー4本の画像案

| ページ | 画像テーマ |
| --- | --- |
| 実家の片付け | 親の家で荷物を3分類している親子 |
| 売れるものの整理 | 段ボールに古い本、カメラ、レコード、ブランド品が並ぶ |
| 買取・処分の進め方 | 買取、処分、確認中の箱に荷物を分ける |
| 家族の悩み | 親子が古い物を見ながら落ち着いて話し合う |

## 画像サイズ

WordPressのアイキャッチは、まず以下を基準にします。

- アイキャッチ: 生成時は16:9、最終書き出しは1200 x 675
- アイキャッチ合成用素材: 生成時は16:9、最終書き出しは1200 x 675
- 記事内イメージ: 生成時は16:9、最終書き出しは1200 x 675
- 写真風素材: 生成時は16:9、最終書き出しは1200 x 675
- 記事内図解: 生成時は16:9、最終書き出しは1200 x 675
- 図解用小物素材: 生成時は1:1、最終書き出しは1024 x 1024
- SNS流用を考える場合: 生成時は1:1、最終書き出しは1200 x 1200

画像生成AIへの指示ではピクセルサイズを厳密指定しません。ピクセルサイズは生成後のリサイズ・書き出し基準として扱います。

## アイキャッチのブランド型

セイリ部のアイキャッチは、以下の型で統一します。

- 参考画像: `assets/images/brand/references/seiribu-brand-reference-01.png` から `seiribu-brand-reference-04.png`
- 背景: 淡いクリーム、淡いミント、薄いグレーなどの明るい無地または薄い生活背景
- ロゴ: Canvaで左上に小さく配置
- 見出し: Canvaで画面上部中央に大きく配置
- サブタイトル: Canvaで見出し直下に小さく配置
- ビジュアル: 画面下部から中央にかけて、人物や実家の物を大きく配置
- 文字パネル: 必要な場合だけ後工程で薄い半透明の四角を使う。基本は大きな白カードで囲わない
- フォント: 見出しはUDモトヤアポロ太字、サブタイトルはNoto Sans JP Regular。画像生成AIにはフォントを指定しない
- フォント未導入時: 制作時はNoto Sans JP Boldなどで暫定制作し、最終Canvaまたは制作環境でUDモトヤアポロ太字へ差し替える
- 見出し色: 記事テーマに合わせて濃紺、濃いえんじ、濃いグレーを使う
- 強調色: 必要な語だけ赤やえんじで強調する
- 避けること: 情報カード風にしすぎる、余白を詰めすぎる、本文図解のように説明過多にする

## ロゴ配置

完成版として公開・納品する画像には、セイリ部ロゴを小さく入れます。アイキャッチのタイトル、サブタイトル、ロゴはCanvaで人間が仕上げます。本文画像・本文図解は、盗用対策、Google画像検索、Pinterest露出時のブランド帰属のため、後工程でロゴを必ず小さく配置します。

- ロゴ元画像: `assets/images/brand/seiribu-logo.png`
- 基本位置: アイキャッチでは左上、本文画像・図解では本文や図解ラベルを邪魔しない右下または左上の余白
- サイズ目安: アイキャッチでは横幅180〜240px程度、本文画像・図解では横幅150〜190px程度
- 余白: 画像端から24〜48px程度あける
- 背景別: 薄い背景では透過ロゴを使い、濃い背景や複雑な背景では薄い白背景を敷いたロゴを使う
- 避けること: 見出し、図解ラベル、人物の顔、重要な品物の上に重ねない
- 画像全体の主役は記事内容にし、ロゴは小さなブランド署名として扱う

ロゴ、タイトル、サブタイトル、日本語ラベルは画像生成AIに描かせません。アイキャッチではCanvaを標準の最終仕上げにします。本文画像・本文図解では画像生成で温かいイラストベースを作り、短い日本語ラベルとロゴはPillow、SVG、Canvaなどの後工程で正確に重ねます。

ユーザーが「Canvaで整える」「背景なしの画像素材だけ」と指定した場合も、素材画像にはロゴ、タイトル、不要な文字を入れません。アイキャッチ完成版ではCanvaの手動作業で背景、文字、ロゴを配置します。本文画像・本文図解ではCodex側の後工程でロゴを配置した完成画像まで作ります。

## 画像生成AIの標準画風

- 温かみのあるフラット編集イラスト
- 主線なし、または控えめな主線
- ベクター風のすっきりした形
- 淡い色、自然光、清潔だが少し生活感のある日本の住まい
- 写真風は、品物の実物感が重要な記事に限定する

画像生成AIには、文字、日本語ラベル、英字、数字、ロゴ、透かし、看板、吹き出しを描かせません。

## 展開・派生の考え方

制作エンジンの初回プランでは、各ピラー1枚ずつ、またはアイキャッチと本文代表画像1枚までに絞るところから始めます。

その後、品目別記事が増えてきたら、ジャンルごとに画像を追加します。

- 本、専門書
- カメラ、レンズ
- レコード、CD
- ブランド品
- 切手、古銭
- フィギュア、おもちゃ
- 工具、楽器

派生画像は、同じブランドトーンを保ちながら記事固有の主役を変えます。同じ人物配置、同じ箱分け、同じ背景を別記事へ貼り替えるだけの流用はしません。

## 制作物の状態管理

- `source`: 生成直後や背景透過前の元素材
- `draft`: 採用前の候補
- `final`: QAを通過し、記事やWordPressで使う確定画像
- `rejected`: 不採用画像。公開用の `assets/images/...` には残さず、必要なら `/private/tmp` など作業用の場所へ退避する

READMEや画像制作プランには、`final` として使う画像だけを掲載します。試作や不採用画像を公開フォルダに混ぜないようにします。

imagegenで作る候補画像は、原則 `/private/tmp/seiribu-image-work/<article-slug>/` に置きます。日本語ラベル入りの図解や比較表は `visualize`、Pillow、SVG、Canvaなどで仕上げ、同じく一時置き場で確認してから採用画像だけを `assets/images/<article-slug>/` に移します。

過去候補が公開用フォルダに貯まってきたら、まず以下で退避候補を確認します。

```bash
python scripts/clean_image_assets.py --article <article-slug> --dry-run
```

確認後、実際に退避する場合だけ `--apply` を付けます。退避先は `/private/tmp/seiribu-image-archive/` で、ファイルは削除しません。複数記事をまとめて退避する場合は、ドライランで確認したあとに限り `--all --apply` を使います。
