# セイリ部 編集部システム作業ルール

## 目的

このリポジトリは、セイリ部の記事を無制限に量産するためではなく、編集部の判断・確認・改善を支えるための作業場です。

現在のCodexは、記事本文を書くエージェントとしては使いません。

記事本文の生成は精度が安定せず、チャットも重くなりやすいため、この作業場では主に以下を担当します。

1. サイト運営の進捗管理
2. 完成した記事に使う画像制作・画像指示の作成
3. 記事ラインナップ、カテゴリ、グロナビ、内部リンク方針の管理
4. 公開前チェック、改善タスク、Notionタスク化の補助

記事本文、導入文、H2本文、FAQ本文の新規生成や全文リライトは、ユーザーが明示的に依頼した場合を除き行いません。

公開判断と最終責任は必ず人間が持ちます。

詳細な運用方針は `strategy/agent-operating-mode.md` を参照します。

## サイトコンセプト

セイリ部は、実家の片付けや持ち物整理で出てきたものを「売る・残す・捨てる」に分けるための整理メディアです。

中心メッセージ:

捨てる前に、売れるか確認。

## 短期の勝ち筋

ゼロイチ検証では、いきなり商標記事や口コミ記事を量産しません。

過去の流入実績では、「急に物を減らしたくなる心理」「実家の物が多すぎてイライラ」「物を捨てられない男性の心理」など、心理・家族の悩み系ロングテールに反応があります。

そのため、短期では以下の流れを重視します。

1. 心理・家族の悩み系記事で入口を作る
2. 勝手に捨てない、家族で揉めない考え方を示す
3. 売る・残す・処分する判断へつなぐ
4. 価値がありそうな物は査定・買取判断記事へ送る
5. 実家整理ピラーや品目別記事へ内部リンクする

心理記事はミニマリズムや自己啓発で終わらせず、必ず実家の荷物整理、価値確認、買取・処分判断へ接続します。

## 読者

主読者:

- 親の家・実家の荷物を片付けている40から60代
- 実家で出てきた古い物を捨てていいか迷っている人
- 買取不可、処分費、親との揉めごとに不安がある人

副読者:

- 50代以上の男性
- コレクションや趣味品を家族に迷惑をかけず整理したい人

## 文体

- です・ます調
- やさしいが、甘すぎない
- 断定しすぎない
- 中学生でも読める言葉を使う
- 業者っぽい煽りを避ける
- 「高く売れます」と断定しない
- 「売れる可能性があります」「査定対象になることがあります」と表現する

## 禁止表現

- 絶対に高く売れる
- 必ず買取される
- どんな状態でも売れる
- 今すぐ申し込まないと損
- ランキングだけで根拠なくおすすめする
- 実体験がないのに「使ってみた」と書く
- 根拠のない相場金額を書く

## 記事タイプ

### 品目別記事

古い品物、実家で見つかるもの、コレクション品などについて、売れる可能性、売れにくい条件、処分前の確認事項を整理します。

基本構成:

1. この記事でわかること
2. 古い品物は売れる可能性があるか
3. 売れる可能性があるもの
4. 売れにくいもの
5. 捨てる前に確認するポイント
6. メルカリ向きか、買取業者向きか
7. 買取不可だった場合の処分方法
8. よくある質問
9. まとめ
10. 関連記事への内部リンク

### 悩み解決記事

「買取基準に満たない」「親が捨てさせてくれない」「処分費が不安」など、片付け時の困りごとに答えます。

心理・家族の悩み系記事では、共感だけで終わらせません。読者が次にできる行動として、写真を撮る、本人に確認する、残す物を分ける、売れそうな物を査定に出す、処分前に確認するなど、実務の一歩を入れます。

### 比較・判断記事

「買取と回収どちらが先か」「メルカリと買取業者どちらが向くか」など、読者が次の行動を選べるように整理します。

## 記事制作フェーズ停止ルール

主力記事、ピラー記事、既存流入記事、収益に近い記事は、一括生成しません。

Codexは以下のフェーズを守り、各フェーズの最後で必ず停止して人間の確認を待ちます。ユーザーの「OK」「次へ」「続けて」などの明示がない限り、次フェーズへ進みません。

### 記事制作ゲート

セイリ部の記事制作、リライト、構成レビューでは、`seiribu-seo-editor` スキルを使います。共有用のスキル本体は `.codex/skills/seiribu-seo-editor/` に置きます。

AGENTS.mdでは最低ルールだけを保持し、詳細な実行手順はスキル側に置きます。

- 企画・構成フェーズで本文を書かない
- 記事制作の成果物は下書きファイルに保存し、チャットには本文・構成・FAQなどの長文を丸ごと出さない
- 本文作成前に検索意図とトピッククラスタ境界を確認する
- 各H2に役割を付け、主役以外を厚く書かない
- 別トピッククラスタをH2化しない
- 構成承認後は、検索意図、扱う範囲、扱わない範囲、H2役割を下書きファイルに残す
- 「必ず入れる要素」は、検索意図に合わせて表・短い注意書き・内部リンク導線として配置する

### 標準フロー

対象:

- ピラー記事
- Search Consoleで流入がある既存記事
- 心理・家族の悩み系の主力記事
- 買取判断、業者選び、収益導線に近い記事

フェーズ:

1. 記事企画
   - 狙うKW、読者、検索意図、記事の役割、扱わない範囲を決める
   - ここでは本文を書かない
2. 見出し構成
   - H1、H2、H3、内部リンク、CTA、画像候補を決める
   - 構成承認前に本文を書かない
3. 導入文制作
   - 読者の状況、不安、結論、この記事でわかることをまとめる
   - H2-1とは別に作り、本文の繰り返しにしない
   - 導入文の最後で停止し、H2本文へ進む確認を待つ
4. H2ごとの本文作成
   - 1回に書くのは原則1つのH2だけ
   - 各H2の最後で停止し、次のH2へ進む確認を待つ
5. 統合・最終調整
   - 重複、流れ、タイトル、メタディスクリプション、画像ALT、内部リンク、CTAを整える
   - 公開前チェックで停止する

### 軽量フロー

短い補助記事、FAQ、用語記事は、ユーザーが明示した場合のみ、記事企画と見出し構成をまとめて出してよいです。

ただし、以下は必ず守ります。

- 本文作成前に構成を確認する
- 根拠のない断定をしない
- 最終公開は人間が判断する

### 停止時の出力

各フェーズの最後は、次のように確認して止まります。

「この方針で次のフェーズへ進めてよいですか？」

## 必ず入れる要素

- 判断表
- 捨てる前チェックリスト
- 買取不可だった場合の選択肢
- 関連記事への内部リンク
- 広告・アフィリエイト表記が必要な場合の注記

## 内部リンク方針

すべての記事から、文脈に応じて以下の記事へリンクできるか検討します。

- 物を捨てられない人の心理と家族ができる整理の進め方
- 親が物を捨てさせてくれないときの話し方
- 実家の片付けで売れるもの一覧
- 買取基準に満たないと言われた理由
- 実家の不用品は買取と回収どちらが先？
- 買取不可だったものを処分する方法

内部リンクは読者の判断を助ける場所にだけ入れます。無理に詰め込みません。

## frontmatter

記事、構成案、下書きには次のfrontmatterを付けます。

```yaml
---
title:
slug:
category:
target_query:
search_intent:
article_type:
status: draft
affiliate_intent: low
human_review_required: true
---
```

## 品質チェック

公開前に必ず以下を確認します。

- タイトルと本文が一致している
- 読者の検索意図に答えている
- 根拠のない断定がない
- アフィリエイト誘導が強すぎない
- 内部リンクが自然に入っている
- 同じ内容の記事と重複していない
- 一次情報がない部分は断定していない
- `human_review_required: true` が残っている

## 作業時の出力先

- Search Consoleなどの入力データ: `data/`
- サイト方針: `strategy/`
- 記事企画: `briefs/`
- 下書き: `drafts/`
- 人間レビュー後の原稿: `reviewed/`
- レポート: `reports/`

## 公開ルール

CodexはWordPressの下書き投入までを支援してよいですが、公開操作は行いません。公開ボタンは人間が押します。
